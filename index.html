Framework to manage Complexity

PCAP

Primitives

They are the building blocks of a system like +, - , *.

Components

They are the things/combinations you make of those primitives. For eg. -  if, While etc.

Abstractions

This is where you name your combinations and forget about them. For eg. - def a function and reuse it again and again.

Patterns

This is a regular and intelligible form or sequence discernible in the way in which something happens or is done. For eg. -  High order function that takes in a function and returns a function

In software engineering lure, we often hear about "breaking down the problem into manageable chunks", this is just that. Composing problems into manageble chunks. Stop thing about things that have already been done. Create on top of what you have already built.
